        1 设计目标与边界
            1.1 功能目标
                在 Web 端（尤其手机浏览器）通过摄像头扫描图书背面条码，或手动输入 ISBN，
                经格式校验后加入「我的书库」；为后续自动爬取元数据、入库提供合法 ISBN。
            1.2 系统边界
                仅处理标准图书条码（EAN-13 图书类，常见以 978/979 开头；或 ISBN-10）。
                不依赖独立 App，使用浏览器能力（HTTPS 或 localhost 下摄像头权限）。
            1.3 展示形态
                PC 与移动浏览器同一套页面（Vue 响应式）；扫码在窄屏设备上为主场景。

        2 前端模块设计
            2.1 入口与页面
                路径：登录后 → 图书列表（`/books`）→ 点击「添加图书」打开对话框。
            2.2 交互组件
                A. ISBN 输入框（Element Plus `el-input`）
                    - 支持带横杠、空格的粘贴；失焦时触发服务端校验。
                    - 展示校验结果：格式正确 / 格式不正确（支持 ISBN-10 / ISBN-13）。
                B. 「扫码」按钮（`el-input` 右侧插槽）
                    - 打开全屏对话框 `IsbnBarcodeScanner.vue`。
                C. 扫码对话框
                    - 动态加载 `vue3-barcode-qrcode-reader`，调用 `getUserMedia` 打开摄像头。
                    - 识别到条码后解析原始字符串，抽取 ISBN：
                        NFKC 归一化全角数字、兼容 14 位 GTIN（前导 0）、长串中提取 978/979 段。
                    - 将抽取的 ISBN 写回表单，并再次请求 `/api/isbn/validate` 同步「格式正确」状态。
                D. 「确定添加」
                    - 若未校验则先调校验接口；通过后 POST `/api/books/add` 传入 `{ isbn }`。
                    - 超时时间单独延长（爬取多数据源可能较慢）。
            2.3 状态与数据流（前端）
                addForm.isbn、isbnValidation.valid（null | true | false）
                扫码成功 → 写入 isbn → 异步 validate → 更新 valid
                提交 → 校验 → addBook → 关闭弹窗 → 刷新列表

        3 后端模块设计
            3.1 接口一览
                | 方法 | 路径 | 说明 |
                | POST | /api/isbn/validate | 校验 ISBN，返回 valid、normalized |
                | POST | /api/books/add | 接收 isbn，查重 → 爬取 → 入库 → 可选 AI 打标签 |
            3.2 ISBN 处理逻辑（utils/isbn_validator.py）
                - Unicode NFKC 归一化（全角数字等）。
                - 仅保留 0-9、X；支持 14 位全数字且首位为 0 时收缩为 13 位。
                - 长度大于 13 时正则提取 `(978|979)\d{10}`。
                - ISBN-10 / ISBN-13 校验位算法验证；通过后返回规范化字符串供入库。
            3.3 入库前置条件
                当前用户下该 ISBN 不存在；ISBN 校验通过；爬取模块能返回有效书名等。

## 附录：ISBN 手机扫码与添加图书 — 使用说明

以下说明对应上文「功能模块设计 · 第一部分」已实现能力，按步骤操作即可。

### 一、环境准备

1. 启动后端（默认 `http://localhost:5000`）与前端（默认 `http://localhost:3000`），方式见《启动说明.md》。
2. 使用 **Chrome / Edge / Safari** 等现代浏览器；手机扫码建议用 **HTTPS 环境或 localhost**（部分浏览器在非安全上下文中禁止摄像头）。
3. 若使用手机与电脑不在同一台机器：需将前端、后端部署到可局域网访问的地址，并保证前后端 CORS 与 Cookie 配置允许你的访问域名（开发阶段通常在同一台电脑上用 `localhost:3000` 即可）。

### 二、登录系统

1. 浏览器打开前端地址（如 `http://localhost:3000`）。
2. 使用已注册账号登录；未注册可先「注册」再登录。

### 三、手动输入 ISBN 添加

1. 进入 **图书列表** 页面（登录后默认或侧栏「图书」）。
2. 点击 **添加图书**。
3. 在 **ISBN** 输入框中输入或粘贴条码下方的数字（可带横杠、空格；支持全角数字，系统会自动归一化）。
4. 点击输入框外或按需要触发校验：提示 **格式正确** 后方可稳定提交。
5. 点击 **确定添加**，等待爬取与入库（可能需十余秒至更久）。
6. 成功后对话框关闭，列表中出现新书；若 ISBN 已存在，会提示图书已存在。

### 四、手机（或电脑）摄像头扫码添加

1. 同样打开 **图书列表** → **添加图书**。
2. 点击 ISBN 输入框右侧 **扫码** 按钮（摄像头图标）。
3. **首次使用**：浏览器会请求摄像头权限，请选择 **允许**。
4. 将手机或笔记本摄像头对准图书背面 **ISBN 条形码**（EAN-13），保持清晰与光照充足。
5. 识别成功后对话框自动关闭，ISBN 自动填入输入框；请确认界面显示 **格式正确**（系统会再次向服务器校验）。
6. 点击 **确定添加** 完成入库，步骤与手动输入相同。

### 五、常见问题

| 现象 | 处理建议 |
|------|----------|
| 无法打开摄像头 | 检查浏览器权限；尝试 HTTPS 或 localhost；改用 **手动输入 ISBN**。 |
| 提示 ISBN 格式不正确 | 确认是否为图书 ISBN（多为 978/979 开头 13 位）；尝试去掉多余前缀后重新校验。 |
| 添加失败 / 网络错误 | 确认后端已启动；查看浏览器控制台与后端终端报错。 |
| 扫码后仍提示格式错误 | 条码污损或误扫非 ISBN；请 **手动输入** 条码数字核对。 |
